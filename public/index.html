The Problem

Nobody built this for
Bitcoin families.

Every existing app either ignores Bitcoin completely, holds your kids' sats in a custodial wallet, or charges you a monthly fee to use your own keys.

01
Fiat apps ignore Bitcoin
Greenlight, BusyKid, GoHenry — all USD, all linked to debit cards. No sats. Teaching kids to use the same broken financial system we're trying to escape.
02
Bitcoin apps charge subscriptions
The only live Bitcoin chore app gates self-custody wallets behind a paid subscription. You pay monthly to use your own keys. That's backwards.
03
Nobody designed for the kid
Every app is built for the parent's dashboard. The child's experience is an afterthought. No kid gets excited to open a dark navy fintech UI.
04
Closed source means blind trust
Bitcoin families are self-sovereignty minded. Asking them to trust a closed-source app with their kids' sat balances is a fundamental values mismatch.
How It Works

Simple by design.
Powerful by default.

BitKids is a coordination layer — not a wallet, not a bank. It connects the work kids do to the Bitcoin parents already hold.

1
Parent sets up the family
Create an account, add child profiles, and input your Bitcoin wallet address. Any wallet works — on-chain or Lightning.
2
Kids get invited
Children create their profile and add their own Bitcoin address. Their sats go directly to their wallet — not a custodial account.
3
Parent creates tasks & sets sat values
Recurring chores, one-off tasks, and learning assignments with YouTube links. Every task gets a commission in sats.
4
Kid taps Done ⚡
Kids see their tasks on a dashboard built for them — colorful, motivating, and designed to feel like a game worth opening.
5
Parent verifies & sends sats
Parent gets notified, reviews the task, opens their wallet app, sends sats directly to the kid's address. BitKids never touches the funds.

Core Principles

Non-negotiable
from day one.

These aren't marketing bullets. They're architectural constraints that shape every product decision we make.

Non-custodial always
BitKids never holds funds. Wallet addresses are stored strings. All payments happen in your own wallet app.
Parent as final authority
No automatic payments. Every sat transfer requires human review and approval. Trust, but verify.
Free forever, no exceptions
No subscriptions. No freemium tiers. No features held hostage. The full product is free for every family, always.
Open source, fully auditable
Every line of code is public on GitHub under MIT license. Read it, audit it, fork it, contribute to it.
Designed for kids, not just about kids
The child's UI must feel like something a kid actually wants to open — not a dashboard their parent lets them see.
Commission, not allowance
Kids earn Bitcoin by doing real work. This is proof-of-work parenting — value exchanged for effort.
